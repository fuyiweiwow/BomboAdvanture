# 炼药系统设计文档

## 一、核心概念

### 1.1 材料维度
每种炼金材料具备以下维度：
- **元素亲和度** (`element`)：`{ fire: 0.4, earth: 0.3 }`，归一化到 0~1
- **可研磨性** (`grindable`)、**最大研磨次数** (`max_grind`)
- **溶剂兼容性** (`solvent_compat`)
- **稀有度等级** (`rarity_level`)：1~5
- **浓度基础值** (`concentration`)：默认为 0.3

### 1.2 研磨系统
- 鼠标拖拽绕研钵一圈 = 1 次研磨
- 每研磨一次颗粒度 +1，同时浓度放大系数增加 (×1.15/次)
- 颗粒度影响最终药剂的元素亲和计算
- 每种材料有 `max_grind` 上限（如草药最高5次，矿物最高6次）
- 不可研磨的材料直接跳过研钵

### 1.3 溶剂系统
溶剂提供：
- **浓度稀释系数** (`concentration_dilution`)：0~1，影响最终浓度
- **元素亲和** (`element_affinity`)：给最终配方额外注入元素

### 1.4 配方系统
配方定义一个目标参数区间（不是硬编码材料列表）：

```json
{
  "condition": {
    "solvent": "water",
    "granularity": { "min": 1, "max": 3 },
    "concentration": { "min": 0.5, "max": 1.5 },
    "element": {
      "fire": { "min": 0.5 },
      "earth": { "min": 0.2 }
    }
  },
  "output_item": { ... }
}
```

匹配规则：玩家投入的材料+研磨+溶剂计算出综合参数，落在某配方的所有区间内即为成功。

### 1.5 配方发现机制
- 初始玩家只有从宝箱/商店获取的配方
- 首次炼出有效组合（参数合理且无已知匹配）→ 自动生成新配方并加入配方书
- 失败不提示原因（只显示失败）

### 1.6 概率失败
- 失败率 = `max(0, 5% × 材料最高稀有度 − 2% × 炼药等级)`
- 稀有材料越多越容易失败（对应高价值药剂）
- 等级越高越稳定
- 严重失败（>=3级）会降低炼药等级

### 1.7 品质分级
| 品质 | 难度分 | 出处 |
|------|--------|------|
| 普通 | 1-5 | 商店 |
| 优秀 | 6-12 | 商店/宝箱 |
| 稀有 | 13-25 | 宝箱 |
| 史诗 | 26-45 | 精英/Boss |
| 传说 | 46+ | Boss |

难度分 = `Σ(材料稀有度×用量) × 复杂度系数`
复杂度系数 = `1 + (研磨次数/3)×0.5 + (维度数-1)×0.3`

## 二、系统结构

### 2.1 数据层
```
assets/alchemy/
├── solvent/          # 溶剂定义（water, oil, venom）
├── recipe/           # 配方文件（暂空，动态生成）

assets/item/*.json    # 材料扩展：+alchemy 字段
```

### 2.2 运行时层
```
src/alchemy/
├── alchemy_data.gd      # 数据加载、参数计算、配方匹配、概率判定
├── recipe_generator.gd  # 随机配方生成、品质评分、从参数反推配方
├── alchemy_lab.gd       # 炼金实验室UI（研磨+坩埚+配方书）
├── recipe_editor/       # [TODO] 配方编辑器
└── alchemy_test.gd      # [TODO] 玩法测试界面
```

### 2.3 参数计算流程
1. 遍历坩埚中的每种材料 × 数量
2. 对每种材料：元素亲和度 × 浓度 × 研磨放大系数 × 溶剂稀释
3. 累加所有元素值
4. 归一化元素值到 0~1
5. 加权计算平均颗粒度
6. 返回 { element, granularity, concentration, solvent }

### 2.4 配方匹配流程
1. 计算参数
2. 遍历已知配方，匹配 condition 的所有区间
3. 如匹配失败但参数看起来合理（浓度>0.3、有元素>0.25）
   → 自动生成新配方（配方发现）
4. 如匹配成功 → 概率判定 → 产出药剂

## 三、UI 交互

### 3.1 炼金实验室布局
```
┌─ HEADER ──────────────────────────────────────┐
│ 炼金实验室                炼药等级: 1    返回 │
├──────┬───────────────────┬────────────────────┤
│ 材料 │     研钵          │     坩埚           │
│      │   ┌───────────┐   │   ┌───────────┐    │
│红草10│   │  磨碎材料   │   │   │  混合区域   │    │
│蓝草10│   │  ○ 杵      │   │   │           │    │
│史莱姆│   │  │         │   │   │ 火草,冰晶  │    │
│蝙蝠翼│   │  颗粒度███ │   │   │           │    │
│火花10│   └───────────┘   │   └───────────┘    │
│冰晶10│     拖拽绕圈研磨   │    溶剂选择         │
├──────┴───────────────────┴────────────────────┤
│ [配方书]  [清空坩埚]  [⚗开始炼药]  状态栏     │
└───────────────────────────────────────────────┘
```

### 3.2 操作流程
1. 左键点击材料 → 放入研钵（可研磨材料）或直接入坩埚
2. 研钵中左键拖拽绕圈 → 研磨，每次一圈 = 1次
3. 右键研钵 → 移入坩埚
4. 右侧选择溶剂
5. 点击「开始炼药」→ 计算判定
6. 成功 → 显示药剂名 + 记录配方
7. 失败 → 仅反馈失败

## 四、配方编辑器

### 4.1 功能需求
- 浏览所有已生成配方
- 编辑配方条件（溶剂、颗粒度区间、浓度区间、元素阈值）
- 编辑产出药剂效果（品质、效果类型、数值）
- 配方评价系统

### 4.2 配方评价系统
- **可行性检测**：给定配方条件，检查是否有材料组合可以达成
- **合理区间建议**：根据现有材料库，推荐可行的参数范围
- **逆天标记**：如果效果数值超过当前材料能达成的合理上限的 200%，标记为"逆天"并警告
- **自动修正**：提供一键修正到合理值的功能
- **拒绝选项**：开发者可以忽略所有建议，强行保存逆天配方

### 4.3 评价算法
```
合理性评分 = 0~100
- 材料可达性（50分）：是否有材料的元素匹配能达到条件
- 浓度可行性（20分）：浓度区间是否在单次炼药可达范围内
- 研磨可达性（15分）：颗粒度是否 ≤ 材料 max_grind
- 效果平衡性（15分）：效果数值与品质等级是否匹配

若单项得分 < 30%，标记对应警告
若总分 < 50，标记"建议修改"
若总分 < 30 且开发者仍保存，标记"逆天"
```
