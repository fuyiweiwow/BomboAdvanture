## 4. 关键结论 / 发现
- **无头模式限制（重要）**：`godot --headless` 使用 dummy renderer，**既不跑 `_process` 也不调用 `_draw`**（4294 行日志全部是启动期 hero 加载错误，回溯止于 `_ready`/`load_hero`，无一止于 `update`/`frame_step`）。因此「绘制 / 逐帧行为」只能在编辑器 F5（play 模式）中验证。
- **`_m` 蒙版错误非致命**：控制台出现的 `cloth10701_stand_m_1_0.png` / `Hair10701…_m…` 加载错误，是角色着色用蒙版 PNG 在原 assets 中不存在，加载器以 `continue` 跳过，**不是灰屏原因**。
- **`dbg.txt` 写入 bug 已修复**：原诊断代码用 `FileAccess.open(path, READ_WRITE)` 在文件不存在时返回 `null`，导致首条 `MAIN_READY` 静默失败、文件从不创建（造成「看不到 dbg.txt」的假象）。已改为：用 `res://dbg.txt`（工程根、编辑器内可写），先读后 `WRITE` 写回，确保文件被创建。
