# Bombo Adventure — 开发计划与设计文档

## 一、项目定位

以**城市为中转舞台**的 **Roguelike RPG** 冒险游戏。

---

## 二、阶段目标

### 阶段一：核心玩法循环

玩家以城市为据点，在以下设施中成长：

| 设施 | 功能 |
|------|------|
| **冒险工会** | 接取随机刷新的任务（根据冒险者等级），完成任务获得奖励并提升等级 |
| **武器店** | 购买/出售武器 |
| **防具店** | 购买/出售防具 |
| **药剂店** | 购买/出售药水 |

#### 关卡系统
- 任务地点为各种场景（关卡），包含：
  - **固定地图** — 手工设计的关卡
  - **程序生成地图** — 每次进入随机生成，支持 Roguelike 玩法
- 关卡掉落：
  - 武器、防具、药水
  - 药水制作材料（为阶段二炼药系统做准备）

### 阶段二：炼药系统
- 利用关卡中收集的材料制作药水（规划中）

---

## 三、当前阶段工具链

以下编辑器为开发者准备，目标是为后续系统提供内容生产管道：

| 编辑器 | 用途 |
|--------|------|
| **角色编辑器** | 定义角色外观与属性，为程序生成人形怪物、地图 NPC、装备系统等提供基础 |
| **地图编辑器** | 制作固定地图（先面向开发者，未来考虑开放给玩家）；支持程序生成参数配置，用于 Roguelike 关卡 |
| **关卡编辑器** | 关卡编排，将地图、怪物、掉落等组合为完整关卡 |
| **怪物编辑器** | 程序生成怪物的外观与属性配置；支持在关卡中编辑可生成的怪物列表 |

## 四、技术栈

- **引擎**: Godot 4.6.2 Mono
- **语言**: GDScript
- **组件系统**: 基于 Atlas 的部位独立换装（body / face / cloth / hair / leg / cap / eye / ear / mouth / cladorn / fpack / npack / thadorn / footprint）
- **数据格式**: JSON（角色、地图、关卡、帧动画）
