# 道具系统设计文档

## 一、核心理念

炸弹即一切。**不提供除炸弹外的任何攻击武器**。所有道具围绕炸弹玩法设计，分为四大类：

| 类别 | 功能 | 来源 |
|------|------|------|
| **消耗品** | 即时生效（回血、回蓝等） | 掉落、商店 |
| **增强品** | 永久/临时提升属性（炸弹数、范围、速度、护甲） | 掉落、商店、Boss |
| **材料** | 炼药系统原材料（植物、怪物掉落） | 可破坏方块、怪物 |
| **钥匙** | 开启对应锁门，推进关卡 | 掉落、区域Boss |

## 二、货币

| 名称 | 说明 |
|------|------|
| **金币** | 通用货币，关卡内拾取，武道会赚取，城市商店消费 |

金币仅分布在关卡地面、可破坏方块、怪物掉落。玩家死亡会损失部分金币。

## 四、关卡机制

### 锁与钥匙
- 关卡地图上放置特殊障碍物 **锁门**（LockedDoor）
- 锁门分两种：**普通锁门**（需铁钥匙）、**Boss锁门**（需击杀区域Boss自动开）
- 玩家持有对应钥匙时，触碰门消耗钥匙并开门
- 无钥匙时门不可通过

### 可破坏方块掉落
- 可破坏方块（软砖）被炸毁时有概率掉落道具
- 基础掉落池：金币（60%）、红草（20%）、蓝草（15%）、铁钥匙（5%）
- 部分方块不出任何东西

### 怪物掉落
- 实现 `npc.gd` 的 `gene_gifts` 接口
- 怪物数据中 `gift` 列表定义掉落：`{ "id": "<item_id>", "weight": int, "min": int, "max": int }`
- 掉落物以 `ItemInstance` 形态生成在地图

### 精英/Boss 掉落
- 必定掉落 3-5 个金币
- 概率掉落稀有道具（幽灵披风等）
- Boss 钥匙必定掉落

## 五、拾取与效果系统

### 拾取触发
1. 玩家移动到物品所在格子
2. 调用 `player.if_take_item()` 检测当前格是否有 `ItemInstance`
3. 有则触发拾取

### 效果处理
- **消耗品**: 立即应用效果（回血等），物品消失
- **增强品**: 修改玩家属性（炸弹数、范围、速度、护甲），物品消失
  - 炸弹数、范围、速度：有上限，达到上限后不再增加
  - 护甲：叠加，无上限
- **材料**: 存入背包/材料栏
- **货币**: 金币数增加，显示拾取飘字
- **钥匙**: 存入钥匙栏

### Buff 系统（幽灵披风）
- 拾取后激活限时 Buff
- 持续期间玩家可穿过障碍物（wall_walking = true）
- 到期后恢复正常

## 六、数据格式

### 道具帧 JSON (`assets/frame/item/<id>.json`)
```json
{
  "NAME": "item1",
  "INTERVAL": 200,
  "STAND": {
    "IMG": ["item1_stand_0_0.png", "item1_stand_0_1.png", ...],
    "CX": [2, 2, 2, ...],
    "CY": [2, 2, 2, ...]
  }
}
```

### 道具数据 JSON (`assets/item/<id>.json`)
```json
{
  "id": "hp_potion_small",
  "name": "Small HP Potion",
  "chs_name": "小红药水",
  "type": "potion",
  "rarity": "common",
  "description": "回复 500 点生命值",
  "frame": "item1",
  "buy_price": 50,
  "sell_price": 25,
  "stackable": true,
  "max_stack": 99,
  "effects": { "heal_hp": 500 }
}
```
